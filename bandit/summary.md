****# ملخص

لعبة Bandit هي حرب إلكترونية لتعلم **أساسيات سطر الأوامر في Linux**، بدءًا من الأوامر البسيطة وصولاً إلى المفاهيم الأكثر تقدمًا مثل Cronjobs، والأمان المقيَّد، وGit.

---

## الجزء الأول: أساسيات سطر الأوامر والتنقيب عن الملفات (Levels 0-12)

| المستوى | التحدي الرئيسي | الأوامر المُستخدمة عادةً | ما تعلمته |
| :---: | :--- | :--- | :--- |
| **0** | تسجيل الدخول عبر SSH | `ssh` | أساسيات الاتصال بخادم عن بُعد (`ssh`). |
| **1** | قراءة ملف بسيط | `ls`, `cat` | سرد الملفات (`ls`) وقراءة المحتوى (`cat`). |
| **2** | قراءة ملف باسم يحتوي على شرطة (-) | `cat ./-` | كيفية التعامل مع أسماء الملفات التي تبدأ بشرطة (`-`). |
| **3** | قراءة ملف باسم يحتوي على مسافات | `cat "spaces in this filename"` أو `cat spaces\ in\ this\ filename` | التعامل مع المسافات في أسماء الملفات. |
| **4** | قراءة ملف مخفي في مجلد فرعي | `ls -la`, `cd`, `cat .hidden` | عرض الملفات المخفية (`ls -la`). |
| **5** | إيجاد ملف قابل للقراءة البشرية فقط | `file *`, `head`, `cat` | استخدام أمر `file` لتحديد أنواع الملفات. |
| **6** | إيجاد ملف بمعايير ملكية وحجم معين | `find / -user bandit7 -group bandit6 -size 33c 2>/dev/null` | استخدام أمر `find` للبحث المتقدم بمرشحات متعددة. |
| **7** | استخراج كلمة مرور بجوار كلمة معينة | `cat data.txt | grep millionth` | التجميع (Piping) و استخدام `grep` للبحث عن أنماط نصية. |
| **8** | إيجاد السطر الفريد في ملف | `sort data.txt | uniq -u` | فرز (`sort`) وتصفية السطور الفريدة (`uniq -u`). |
| **9** | استخراج سلسلة قابلة للقراءة البشرية | `strings data.txt | grep '=='` | استخدام أمر `strings` لاستخراج السلاسل النصية من الملفات الثنائية. |
| **10** | فك تشفير Base64 | `cat data.txt | base64 -d` | فك ترميز البيانات المرمزة بـ **Base64** (`base64 -d`). |
| **11** | فك تشفير Rot13 | `cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m'` | استخدام الأمر **`tr`** (translate) لعمليات استبدال الأحرف. |
| **12** | التعامل مع بيانات مضغوطة ومُرمَّزة بالـ Hex | `xxd -r data.txt > data.bin` ثم فحص `file data.bin` واستخدام `gzip -d` و `bzip2 -d` و `tar -xf` | كيفية عكس الترميز السداسي عشر (`xxd -r`) والتعامل مع أنواع مختلفة من الضغط والأرشفة (gzip, bzip2, tar). |

## الجزء الثاني: مفاهيم الشبكات والأمان (Levels 13-20)

| المستوى | التحدي الرئيسي | الأوامر المُستخدمة عادةً | ما تعلمته |
| :---: | :--- | :--- | :--- |
| **13** | تسجيل الدخول باستخدام مفتاح SSH خاص | `ssh -i sshkey.private bandit14@localhost` | استخدام **مفاتيح SSH الخاصة** لتسجيل الدخول، ومفاهيم الأمان المتعلقة بملكية المفاتيح. |
| **14** | استخدام Netcat لقراءة البيانات من منفذ | `echo <password> | nc localhost 30000` | أساسيات أداة **Netcat (`nc`)** للاتصال بالشبكة وإرسال/استقبال البيانات. |
| **15** | استخدام OpenSSL للاتصال الآمن | `openssl s_client -connect localhost:31000 -ign_eof` | كيفية الاتصال بالخدمات التي تستخدم **SSL/TLS** باستخدام `openssl s_client`. |
| **16** | البحث عن المنفذ المفتوح والمشفَّر | `nmap -p 31000-32000 localhost` ثم استخدام `openssl` مع المنفذ الصحيح. | **مسح المنافذ** باستخدام `nmap` والجمع بين أدوات الشبكات. |
| **17** | إيجاد كلمة المرور في ملف يحتوي على اختلافات | `diff readme -` | استخدام الأمر **`diff`** لمقارنة ملفين وإيجاد الاختلافات. |
| **18** | البحث في سجلات الأخطاء | `ls -la`, `cat readme`, `cat .log` | البحث في **سجلات الأخطاء** واستخدام الأوامر الأساسية في مسارات مختلفة. |
| **19** | استخدام برنامج setuid لتشغيل أمر كـ bandit20 | `./bandit20-do cat /etc/bandit_pass/bandit20` | مفهوم **Setuid binary** وكيفية استغلاله لتشغيل الأوامر بصلاحيات مستخدم آخر. |
| **20** | التفاعل مع ثنائي setuid عبر الشبكة | (Terminal 1) `nc -l -p 1234` و (Terminal 2) `./suconnect 1234` | الجمع بين **Netcat** وثنائي `setuid` لإنشاء اتصال محلي ونقل كلمة المرور. |

## الجزء الثالث: المهام المجدولة وأمان النظام (Levels 21-25)

| المستوى | التحدي الرئيسي | الأوامر المُستخدمة عادةً | ما تعلمته |
| :---: | :--- | :--- | :--- |
| **21** | فحص مهمة Cron مُجدوَلة | `ls /etc/cron.d`, `cat /etc/cron.d/cronjob_bandit23` | فحص ملفات **Cronjobs** في نظام Linux لمعرفة كيفية تشغيل المهام تلقائيًا. |
| **22** | استغلال كتابة Cron لملف مؤقت | `cat /usr/bin/cronjob_bandit23.sh` ثم تعديل ملف مؤقت في `/tmp` لتلقي كلمة المرور. | فهم كيف يعمل الـ **Cron** مع الملفات المؤقتة في `/tmp`. |
| **23** | استغلال مهمة Cron تتطلب تجميعًا | `echo <password> > /tmp/a` ثم `cat /usr/bin/cronjob_bandit24.sh` لاكتشاف عملية التجميع المعقدة. | فهم آليات تنفيذ النصوص البرمجية المعقدة وكيفية حقن الأوامر. |
| **24** | تفاعل الأوامر المجدولة مع المستخدم | **لا يوجد حل مباشر بـ Bash.** يجب استخدام لغة برمجة (مثل Python أو Perl) لإنشاء خادم شبكة يُشغل النص البرمجي. | فهم تحديات التفاعل مع البرامج التي يتم تشغيلها بواسطة المستخدم **bandit25** باستخدام **su**. |
| **25** | هجوم القوة الغاشمة (Brute-Force) عبر الشبكة | `for i in {0000..9999}; do echo "UoMYTrfrz... $i"; done | nc localhost 30002` | تطبيق هجوم **القوة الغاشمة** باستخدام حلقات Bash و **Netcat** عبر منفذ شبكة. |

## الجزء الرابع: الهروب من القيود وأنظمة التحكم في الإصدار (Levels 26-34)

| المستوى | التحدي الرئيسي | الأوامر المُستخدمة عادةً | ما تعلمته |
| :---: | :--- | :--- | :--- |
| **26** | الهروب من بيئة Shell مُقيَّدة (`more`) | **1. تصغير النافذة 2. الضغط على `v` للدخول إلى `vi` 3. داخل `vi`: `:set shell=/bin/bash` 4. `:shell`** | تقنيات **الهروب من الأوامر المُقيَّدة** (Restricted Shell) باستخدام ميزات المحررات التفاعلية مثل `more` و `vi`. |
| **27** | الهروب من `vi` ومحرك `setuid` | **نفس خطوات الهروب من `more` في المستوى 26**، ثم استخدام الملف التنفيذي `bandit27-do` لقراءة كلمة المرور. | دمج تقنيات الهروب مع استغلال ثنائيات `setuid`. |
| **28** | أساسيات Git: نسخ مستودع | `git clone ssh://bandit28-git@localhost/home/bandit28-git/repo` | استخدام **Git** لنسخ المستودعات البعيدة. |
| **29** | Git: فحص سجل التغييرات | `git log`, `git show <commit-hash>` | فحص **سجل التغييرات** (History) للمستودع للعثور على كلمة المرور في إحدى التعديلات السابقة. |
| **30** | Git: فحص الفروع البعيدة | `git branch -a`, `git checkout <branch-name>` | استكشاف **الفروع البعيدة** (Remote Branches) وإجراء سحب عليها. |
| **31** | Git: فحص الوسوم (Tags) | `git tag`, `git show <tag-name>` | فحص **الوسوم** (Tags) في Git التي قد تخزن معلومات حساسة. |
| **32** | Git: تجاهل التغييرات غير الملتزمة (Untracked) | `git status`, `git checkout .` أو `git reset --hard` | كيفية إدارة التغييرات المحلية غير الملتزمة في Git. |
| **33** | فهم متغيرات البيئة | `printenv` أو `export` | فهم كيفية قراءة **متغيرات البيئة** واستخدامها. |
